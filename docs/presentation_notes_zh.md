# AI Pulse Tracker v2 展示材料

## 一句话介绍

AI Pulse Tracker v2 不是普通 AI 新闻聚合器，而是一个面向 AI 产品动态的多 Agent 情报追踪系统，重点提升对海外硅谷一手动态的快反捕捉能力，并基于追踪结果输出趋势判断。

## 演示顺序

### 1. 先看首页

强调四个点：

- 实时感指标卡
- 趋势总览
- Top Signals
- 观察结论卡

### 2. 再看事件卡

重点说明：

- 来源等级 `T0-T4`
- 是否一手
- 新颖度 `novelty_score`
- 为什么被判定为高价值

### 3. 再看来源池和任务状态

这里可以说明这不是静态页面，而是一个有：

- 高频池 / 常规池
- source health
- 分阶段运行状态

的真实追踪系统。

### 4. 最后看日报/周报

说明这个工具不仅能抓，还能把结果转化成可消费的管理摘要。

## 简要分析示例

基于最近公开的一手动态，可以看到 AI 产品竞争正在从“模型能力本身”继续转向“Agent 化执行、开发者入口与多模态延展”。海外硅谷头部公司更新更快，重点更偏向把模型直接封装到工程工作流、搜索、设备与任务链路中，说明竞争焦点已经从单次对话效果，转向是否能进入真实使用流程。国内厂商则继续在开源、平台化和企业承接上同步跟进，意味着下一阶段比拼的不只是模型先进性，也包括交付效率、生态整合与场景落地能力。

## 答辩时可直接说的话

### 关于实时性

“我不是按媒体热度做资讯聚合，而是把官方博客、docs、changelog、GitHub release 和核心负责人披露设成更高优先级的源头池，并通过 3h / 6h / 12h 的实时窗口突出快反能力。”

### 关于 Agent 能力

“我的系统不是简单抓取和摘要，而是把流程拆成了 7 个 Agent：采集、增强、筛选、归并、趋势分析、展示生成、调度主控，每个 Agent 都承担独立判断职责。”

### 关于趋势判断

“趋势分析不是复述新闻，而是先生成 trend snapshot，再给出核心变化、证据事件、背后原因、趋势判断和后续观察点，因此结论有证据支撑。”
